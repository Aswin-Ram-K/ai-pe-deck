### D5 — Scroll engine rewrite: per-frame accumulator (root cause fix for "Next resets to beginning")
**Prompt:** "every time I press Next, the script starts freshly from the very beginning."
**Landed:** Rewrote scrollTick from `baseline + elapsed × speed` to per-frame accumulator `currentOffsetPx += dt × pxPerSec`. Snap = single assignment. Removes the setTimeout+leftover-rAF race that produced the reset. Verified: 7 Next-tap scenarios snap within 1.1 px of target offset. See `PROJECT_CONTEXT.md §7`.

### D6 — Measurement: lazy, gated on layout visibility (root cause fix for "stuck on Waiting for slide 1")
**Prompt:** "it just keeps saying Waiting for slide one, even though slide one has loaded"
**Landed:** `ensureMeasured()` in `teleOnSlideChange` — measures offsets only when `section[data-slide="1"].offsetHeight > 0` (i.e., State B actually visible). On fresh phone load, body has `.on-intro` → State B is `display:none` → all descendants report `offsetTop/offsetHeight === 0`. Lazy measure with rAF retry solves this. Commit `5c1ff36`.

### D7 — Breather + end-of-slide pause
**Prompt:** "After the scrolling comes to the end part of every slide section, I want it to pause… Also, I don't want the scrolling animation to start immediately. Give it a one second breather."
**Landed:** `BREATHER_MS = 1000` gates `scrollTick` via `now < scrollStartAtMs`. End-of-slide: `maxOffsetPx = slideOffsets[N+1] - 12`; when reached, rAF stops scheduling. Commit `dc647e9`.

### D8 — Hold-to-pause + pause button + total/per-slide timers + overtime alert
